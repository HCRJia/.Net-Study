## 1、工作流系统概念

### 1.1、工作流系统理解

含义：用来管理工作流的系统。叫做工作流系统。如下所示，

![img](https://cdn.nlark.com/yuque/0/2026/png/42432244/1769517530153-25fd7a79-cc34-4114-9910-e251ebf34eb6.png)

例如：工作流系统由前端+后端+数据库组成。

前端：HTML CSS JS

后端：ASP.NET Core6.0 WebAPI

数据库：MySQL

### 1.2、工作流系统背景

企业在管理公司的工作上。如果缺乏人手管理，自己一个人管理。时间长了，会出现管理效率低下、非常累的问题。所以，为了提升管理的工作上的效率。使用系统来进行管理。

好处：提升企业管理工作的效率

## 2、工作流设计

### 2.1、工作流理解

含义：工作流程的集合，就是工作流。

![img](https://cdn.nlark.com/yuque/0/2026/png/42432244/1769517571358-5e704739-bd88-4a9b-8059-4ad1c2ee302b.png)

例如：

工作：开发项目

流程：开发项目的过程

如下所示：

![img](https://cdn.nlark.com/yuque/0/2026/png/42432244/1769517571482-192cb0d1-fa39-4c10-b276-46a5a354e8b0.png)

过程： 从项目分析—到—-项目部署。为开发项目过程。

流：把项目过程抽象，成为流。

流属于开发项目。由于开发项目是工作。所以，叫做项目工作流。

### 2.2、工作流类型

#### 2.2.1、简单工作流

含义：工作流中的某个节点，按照序列进行。就是简单工作流。如下所示：

![img](https://cdn.nlark.com/yuque/0/2026/png/42432244/1769517571848-382751f7-d7ca-4921-b3d0-7889b2ac8c56.png)

解释：从项目分析，到项目部署，都是按照顺序依次往下。叫做简单工作流

#### 2.2.2、分支工作流

含义：工作流中的某个节点，出现了分支节点。就是分支工作流。如下所示：

![img](https://cdn.nlark.com/yuque/0/2026/png/42432244/1769517571885-30a50b21-06d5-46df-9a22-b95fa26f1bac.png)

解释：项目开发这个节点上，出现了项目模型设计、项目接口设计这两个分支。

#### 2.2.3、合并工作流

含义：工作流中的某个节点，出现了合并节点。就是合并工作流。如下所示：

![img](https://cdn.nlark.com/yuque/0/2026/png/42432244/1769517572003-53bf8a45-71ad-4dcb-a8bb-eb9e7c7c198a.png)

解释：项目测试这个节点上，出现了对项目模块开发、项目bug开发两个分支合并。

总结：工作流核心，分为2个工作流。简单工作流和复杂工作流。复杂工作流又包括，分支工作流和合并工作流。

### 2.3、工作流设计

含义：包括工作流设计器、工作流模型设计、工作流操作模型设计

#### 2.3.1、工作流设计器

含义：设计工作流的工具。例如：使用设计器画一个工作流。如下所示：![img](https://cdn.nlark.com/yuque/0/2026/png/42432244/1769517571696-54cb2f64-a011-43ff-bca2-bcc4391b1819.png)

#### 2.3.2、工作流模型设计

含义：使用设计器设计出来的工作流。对具体工作流抽象。对工作流本身进行抽象。

目的：方便管理所有的工作流。

如下所示：

```plain
基本抽象如下：
1、工作流模型：WorkFlow
2、工作流节点：FlowNode
	2.1、工作流节点类型：FlowNodeType
3、工作流连接：FlowLine		
4、工作流类型：WorkFlowType

抽象关系如下：
WorkFlow ：
 List<FlowNode>
 List<FlowLine>
 WorkFlowType

总结：由浅入深的抽象。工作流中的每一个点都进行抽象。然后将每个抽象元素形成关系。最后实践。
```

#### 2.3.3、工作流操作模型设计

含义：人会操作工作流中某个节点。对工作流操作进行抽象。例如：工作流执行。工作流执行状态。工作流执行流转、工作流执行状态改变、工作流执行结果、工作流执行菜单、工作流执行指定。操作抽象。

如下所示：

```plain
抽象如下：
1、工作流执行模型：WorkFlowProcess
2、工作流执行状态模型：WorkFlowStatus
3、工作流执行流转模型：WorkFlowProcessTransition
4、工作流执行状态改变模型：WorkFlowStatusChange
5、工作流执行结果模型：WorkFlowResut
6、工作流执行菜单模型：WorkFlowMenu
7、工作流执行指定模型：MakerListModel

第三方框架：
落地框架：JadeFramework.WorkFlow
```
