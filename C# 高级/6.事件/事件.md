# 事件是什么

事件和委托其实是息息相关的，事件其实是对委托的一个约束，事件在委托的基础上提供了更安全的封装机制。

# 事件的使用

事件使用event来声明：

```csharp
public event EventHandler PriceChangedHandler;

public class XEventArgs : EventArgs
{
    public int OldPrice { get; set; }
    public int NewPrice { get; set; }
}

public class Lesson
{
    public int Id { get; set; }
    public string Name { get; set; }

    private int _price = int.MaxValue;
    public int Price
    {
        get
        {
            return this._price;
        }
        set
        {
            if (value < this._price)
            {
                this.PriceChangedHandler?.Invoke(this,
                    new XEventArgs()
                    {
                        OldPrice = this._price,
                        NewPrice = value
                    });
                this._price = value;
            }
        }
    }
```

这里写了个事件的声明，又写了事件的实现。EventHandler是委托类，PriceChangedHandler便是事件的名称。Lesson类中为Price值赋值时写了PriceChangedHandler的具体的使用。

```csharp
public static void Show()
{
    Lesson lesson = new Lesson()
    {
        Id = 123,
        Name = "c#进阶",
        Price = 4699
    };
    //订阅：把订户和发布者的事件关联起来
    lesson.PriceChangedHandler += new Student().Buy;
    lesson.PriceChangedHandler += new Tencent().Popularize;
    lesson.Price = 3999;
}


/// <summary>
/// 订户：关注事件，事件发生后，自己做出对应的动作
/// </summary>
public class Student
{
    public void Buy(object sender, EventArgs e)
    {
        Lesson lesson = (Lesson)sender;
        Console.WriteLine($"This is {lesson.Name} Lesson");

        XEventArgs args = (XEventArgs)e;
        Console.WriteLine($"之前价格{args.OldPrice}");
        Console.WriteLine($"现在价格{args.NewPrice}");
        Console.WriteLine("果断买了！！！");
    }
}
public class Tencent
{
    public void Popularize(object sender, EventArgs e)
    {
        Lesson lesson = (Lesson)sender;
        Console.WriteLine($"This is {lesson.Name} Lesson");

        XEventArgs args = (XEventArgs)e;
        Console.WriteLine($"之前价格{args.OldPrice}");
        Console.WriteLine($"现在价格{args.NewPrice}");
        Console.WriteLine("广大用户请留意！！！");
    }
}
```

# 事件的作用

我们可以看下这个编译器的报错：

![img](https://cdn.nlark.com/yuque/0/2025/png/42432244/1761835748854-1e8a18ca-ce84-41c3-a3e6-59cfc989c530.png)

这里显示PriceChangedHandler只能出现在 += 或 -= 的左边，使用PriceChangedHandler = 方法名 语句是错误的。这就是事件的作用，事件的核心是**发布-订阅模式中的封装性**，具体体现在两个层面：

1. **防止订阅者之间互相覆盖**：如果直接用 `=` 赋值，后一个订阅者会把前一个订阅者覆盖掉；而用 `+=` 则可以叠加多个订阅者，各自独立。
2. **防止外部随意触发事件**：事件只能由声明它的类内部触发（如 `this.PriceChangedHandler?.Invoke(...)`），外部不可直接调用。而普通委托变量，外部代码可以随时 `obj.PriceChangedHandler(...)` 调用，破坏了封装。

委托和事件的核心区别：委托是"方法的容器"，事件是"委托的安全封装"。这也是事件相比委托的优点。
